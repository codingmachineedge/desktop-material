Git, rebuilt in Material Design 3.

Desktop Material is an independent M3 Expressive remake of GitHub Desktop — an icon navigation rail, a floating pill toolbar, browser-like repo tabs, real multi-account, versioned settings, and non-modal dialogs, wrapped in an animated light/dark shell.

Material foundation and shipped roadmap

A Material Design 3 shell with verified native workflows

The core shell and named Git/GitHub workflows are published on main. Cards marked Verified on main have passed the exact production UI gate and are represented in the current screenshot gallery.

🎨

M3 Expressive shell

Animated light/dark theming, soft elevation, an app bar with an inline pill menu, and floating radius-24 workspace cards.

🧭

Icon navigation rail

A left rail for Changes (with a badge), History, Branches, Settings, and your account avatar — plus a floating pill toolbar with repo and branch chips and a sync pill.

🗂️

Browser-like repo tabs

Tabs bound per-account to repositories, with inline rename and per-tab title styling — bold, italic, size, colour, font, alignment.

👥

Real multi-account

Multiple identities per host, each with its own tabs, repos, and settings. Switching the active account switches the whole workspace context.

🕔

Versioned settings

Every settings and tab change auto-commits to a local git repo, with a non-modal history side sheet for diffs, undo, redo, and restore. Each action appends an audit commit instead of rewriting history.

🪟

Non-modal dialogs

Dialogs float without blocking the app, drag by their headers, cascade, and come to front on focus. Preferences is an MD3 940×660 dialog; repo and branch pickers are side sheets.

Power features

Built for real-world Git work

Delivery and verification history is tracked in the project's living roadmap. Git, gh, REST, and GraphQL capabilities become named, guided workflows—not a searchable dump of commands or API endpoints.

🔔

Notification centre Shipped

A Git-backed Local view plus an account-aware GitHub inbox with All, Unread, and participating-only controls and a complete signed-out state.

🔍

Search everywhere Shipped

Filter chips, substring/regex modes, and a full regex builder, plus History search by title, message, tag, or hash and an optional commit ancestry graph.

⧉

Multi-clone & orgs Shipped

Browse complete organization lists, select many repositories, clone in parallel or sequence, import/export URL-only lists, and publish directly into an organization.

↘

Guided repository functions Verified on main

Shallow-clone with a commit-depth field, deepen to a bounded or complete history from Repository Tools, and manage cone-mode sparse checkout with validation and an explicit review step.

⚡

Automation Shipped

Guarded commit/push and pull schedules with global, account, and repository overrides, plus merge-all branches/worktrees with per-target progress and Copilot conflict assistance.

▷

GitHub Actions Verified on main

Filter and retain workflow-run pages, switch attempts, load and retry bounded job pages, inspect logs, re-run the exact job, review pending deployments and history, and separately confirm eligible fork runs. Artifacts retain bounded paging, native download, and local digest context.

PR

Native pull requests Verified on main

Choose the exact repository, account, base, and current head; write a title and Markdown description; review the mutation; and submit or create a draft without entering a command or endpoint.

BR

Effective branch rules Verified on main

Inspect reviews, checks, deployments, merge queue, signatures, history, update/delete/force policy, bypass context, and source rulesets with signed-out and account-selection recovery paths.

🤖

Agent access & CLI Shipped

Opt-in, token-gated MCP and REST on loopback only, backed by a stdio proxy and command-line client for repositories, Git, tabs, automation, and workflow dispatch.

🏢

Provider accounts Shipped

Use multiple GitHub identities, self-hosted GitLab endpoints with PATs, and Bitbucket app passwords; browse and clone each provider's repositories without exposing credentials.

⌁

Provider-neutral triage Integrated

Triage bounded Issue and pull-request summaries for the exact GitHub, GitLab, or Bitbucket account and repository, with safe links and explicit unsupported, partial, capped, and error states.

🧰

Desktop-plus parity Shipped

Multiple inspectable stashes, repository pinning and grouping, pull-all with per-repo results and exact-origin signed-in account fallback for ambiguous HTTPS authentication or not-found responses, branch presets, default-branch controls, and repository editor overrides.

🪟

Multi-window workflows Shipped

Open repositories and worktrees in separate windows with isolated selection, correct command routing, and persisted per-window repository tabs.
